* CSS Grid can be declared by setting the display to grid.
* It is a 2D layout system i.e it positions both rows and columns simultaneously.

* We can define the number and sizes of the columns using grid-template-columns.
* We can define the number and sizes of the rows using grid-template-rows.
* fr is the unit used in grid layouts which means fraction of the available space.

* Unlike FlexBox the items are in the columns direction and wrap by default when there are more items then the columns.

* We can adjust gap between the the rows and columns by using row-gap and column-gap.
* gap is the shorthand for row-gap and column-gap i.e gap: <row-gap> <col-gap>;

* Explicit grid is the grid layout in which we manually decide the number of rows and columns.
* Implicit grid is the grid layout in which we only set either number of rows or columns and the other one is created automatically.

* repeat() can be used to repeat a sequence of row/column values n times. i.e repeat(n, value1 value2 value3 .....)

* minmax(min, max) can be used to restrict the items from being smaller then min and not larger than the max value.

* min-content is the keyword which shrinks the track to the unbreakable size of the word in the track.

* max-content is the keyword which grows the track to the largest sentence in the track.

* fit-content() works the same way as max-content when the text is less than the specified value. But when the text exceeds the specified value the text wraps down to the next line.

* grid-column and grid-row can be used to specify the starting and ending grid line of a grid item.
* It specifies the starting line and the ending line number or specifies the number of rows/columns to span using span keyword.

* grid-area can also be used to specify both grid-column and grid-row in the format row-stat / col-start / row-end / col-end.

* Grid Template Areas can be used to place items by using readable names instead of line numbers.
* grid-area then uses that string name to pace items to its position.
* Each string in grid-template-column represents a row.
* Each word inside the string represent a grid cell.
* The strings are separated by space.
* Each word in the string are also separated by a space.
* The grid-area in the items use that string without enclosing in the quotes.
* Use a period to represent an empty cell.
* Every row in the template area must have the same number of cells.
* The named area must form a rectangle.

* grid-auto-flow can change the filling direction of the cells to row or column.

* grid-auto-rows specifies the size of the auto generated row tracks.

* grid-auto-columns specified the size of the auto generated column tracks.

* justify-content can be used to position the whole grid horizontally inside the grid container if the container is larger than the grid.
* align-content can be used to position the whole grid vertically inside the grid container if the container is larger than the grid.
* place-content is the shorthand for align-content and justify-content.

* justify-items can be used to position the items inside the cells horizontally.
* align-items can be used to position the items inside the cells vertically.
* place-items is the shorthand for align-items and justify-items.

* justify-self can be used on individual grid item to position itself horizontally.
* align-self can be used on individual grid item to position itself vertically.
* place-self is the shorthand for align-self and justify-self.
